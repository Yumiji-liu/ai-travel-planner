## 1. 产品概述

### 1.1 产品愿景

打造一个面向全球用户的 AI 驱动个性化旅行规划平台，让任何人都能在几分钟内获得一份量身定制的完整旅行行程。

### 1.2 核心价值主张

- **省时**：传统旅行规划需要数小时，AI 规划只需 30 秒
- **个性化**：根据用户偏好、预算、出行风格生成专属行程
- **可视化**：行程在地图上直观呈现，一目了然
- **免费**：核心功能完全免费，通过广告变现

### 1.3 目标用户

- **主要用户**：18-45 岁，有出行计划的全球用户
- **语言**：英文优先，后续支持多语言
- **设备**：桌面端优先，响应式兼容移动端

## 2. MVP 功能范围

### 2.1 功能清单（In Scope）

| 优先级 | 功能模块 | 功能描述 |
|--------|----------|----------|
| P0 | 对话式行程生成 | 用户输入目的地、天数、偏好，AI 生成完整行程 |
| P0 | 行程展示页 | 按天展示景点、餐厅、交通安排 |
| P0 | 地图可视化 | 在地图上标注每日路线和打卡点 |
| P1 | 行程导出 | 支持导出为 PDF 或生成分享链接 |
| P1 | 用户账号 | 注册/登录，保存历史行程 |
| P1 | 行程编辑 | 手动调整 AI 生成的行程内容 |
| P2 | 广告位 | 行程页侧边栏展示相关广告（酒店/机票） |

### 2.2 不在 MVP 范围内（Out of Scope）

- 实时机票/酒店价格查询
- 多人协作规划
- 移动端 App（iOS/Android）
- 社区/UGC 内容
- AI 学习用户历史偏好
- 支付功能

## 3. 用户故事

### 3.1 核心流程（Happy Path）

**作为一个想去日本旅行的用户，我希望：**

1. 打开网站，看到简洁的输入界面
2. 填写：目的地（东京）、天数（7天）、出行风格（文化体验+美食）、预算（中等）
3. 点击"生成行程"，等待约 10-15 秒
4. 看到一份完整的 7 天行程，包含每天的景点、餐厅推荐和大致路线
5. 在地图上看到每天的路线可视化
6. 对某天的安排不满意，可以手动修改或让 AI 重新生成该天
7. 满意后导出 PDF 或复制分享链接

### 3.2 用户故事列表

**US-001：行程生成**
- 作为用户，我可以输入目的地、出行天数、人数、偏好标签，获得 AI 生成的行程
- 验收标准：生成时间 ≤ 20 秒；行程包含每天至少 3 个活动；每个活动有名称、简介、建议停留时长

**US-002：地图展示**
- 作为用户，我可以在地图上看到行程中所有地点的标注和每日路线连线
- 验收标准：地点标注准确；支持点击标注查看详情；支持按天筛选显示

**US-003：行程编辑**
- 作为用户，我可以删除、替换或重新排序行程中的某个活动
- 验收标准：编辑后地图实时更新；支持"撤销"操作

**US-004：导出与分享**
- 作为用户，我可以将行程导出为 PDF 或生成一个可分享的只读链接
- 验收标准：PDF 格式清晰可读；分享链接无需登录即可查看

**US-005：用户账号**
- 作为用户，我可以注册账号并保存我的行程，下次登录后可以继续查看和编辑
- 验收标准：支持 Google OAuth 登录；行程列表按时间排序

## 4. 技术方案

### 4.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | Next.js 14 (App Router) | SSR + SEO 友好 |
| UI 样式 | Tailwind CSS + shadcn/ui | 快速开发，风格统一 |
| 地图 | Mapbox GL JS | 免费额度充足，样式美观 |
| 动效 | Framer Motion | 页面过渡和加载动效 |
| 后端 | Next.js API Routes | MVP 阶段无需独立后端 |
| AI | OpenAI GPT-4o / Claude API | 行程生成核心 |
| 数据库 | Supabase (PostgreSQL) | 用户数据 + 行程存储 |
| 认证 | Supabase Auth + Google OAuth | 快速集成 |
| 文件存储 | Supabase Storage | PDF 导出存储 |
| 部署 | Vercel | 与 Next.js 无缝集成 |
| 广告 | Google AdSense | 免费接入，全球覆盖 |

### 4.2 系统架构（简化）

```
用户浏览器
    │
    ▼
Next.js (Vercel)
    ├── 前端页面 (React)
    │       ├── 输入表单
    │       ├── 行程展示
    │       └── 地图组件 (Mapbox)
    │
    └── API Routes
            ├── /api/generate  → 调用 AI API 生成行程
            ├── /api/trips     → CRUD 行程数据
            └── /api/export    → 生成 PDF
                    │
                    ▼
            Supabase (DB + Auth + Storage)
```

### 4.3 AI Prompt 设计思路

行程生成 Prompt 需包含：
- 目的地、天数、人数、预算等级、偏好标签
- 输出格式要求（JSON 结构化数据）
- 每个活动字段：名称、地址、坐标、类型、简介、建议时长、开放时间

|--------|------|------|------|

## 11. 风险与应对

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|----------|
| AI API 成本超预期 | 中 | 高 | 设置每日调用上限；缓存相似行程结果 |
| AI 生成内容不准确 | 中 | 中 | 加入用户反馈机制；Prompt 持续优化 |
| Mapbox 免费额度不足 | 低 | 中 | 监控用量；备选 OpenStreetMap |
| Google AdSense 审核未通过 | 低 | 中 | 提前申请；备选联盟广告 |
